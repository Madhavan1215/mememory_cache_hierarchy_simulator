🔴 The Problem — No Optimisation

Normal CPU, L1 holds only 4 items. Watch Cold Misses and Evictions force expensive 100-cycle Storage trips every time.

The Memory Wall
Storage is 100× slower than L1.
Cold Miss = first time seeing an address — always goes to Storage.
Eviction = L1 full, oldest item thrown out.
This gap is what every AI chip tries to close.

🔵 Solution 1 — Bigger Cache

L1 expanded from 4 → 8 slots. More items fit, fewer evictions, repeated addresses stay warm longer.
Real world: Google TPU has 144MB on-chip SRAM.

Why It Helps
Bigger L1 = fewer evictions.
Fewer evictions = repeated data stays in L1.
Like upgrading from a small pocket to a big bag — you carry more so you go to the store less.

🟡 Solution 2 — Prefetching

While using address X, secretly load the NEXT address into L1 early. By the time it is needed — already there.
Real world: iPhone chip predicts and preloads data.

🟢 Solution 3 — Data Reuse

Sort addresses so same ones are grouped. Each address only suffers one Cold Miss then stays in L1 for all its accesses.
Real world: Google TPU tiles workloads to reuse weights.
